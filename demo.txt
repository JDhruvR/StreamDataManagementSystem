# SDMS Demo Guide (Normal + Stream-Stream Join)

This file gives an exact demo flow with minimal setup friction.

## 1) One-time prerequisites

1. Create and activate virtual env:
```bash
python -m venv venv
source venv/bin/activate
```

2. Install dependencies:
```bash
pip install -r requirements.txt
```

3. Ensure Kafka exists under:
```text
kafka_2.13-3.6.1/
```

4. Make launcher executable:
```bash
chmod +x demo.sh
chmod +x demo_cleanup.sh
```

---

## 2) Fast demo launcher (tmux-based)

The script starts everything in a tmux session named `sdms-demo`.

### A) Normal demo (stream -> table join)
```bash
./demo.sh normal
tmux attach -t sdms-demo
```

In the CLI pane, run:
```text
load schemas/pollution2.json
status
```

Then create/populate the reference table used by join query:
```text
table_create
table_name> sensors
columns> id:STRING,name:STRING

table_insert
table_name> sensors
row_json> {"id":"s1","name":"sensor_1"}

table_insert
table_name> sensors
row_json> {"id":"s2","name":"sensor_2"}

table_insert
table_name> sensors
row_json> {"id":"s3","name":"sensor_3"}
```

Watch output in consumer pane on topic `pollution_out`.

---

### B) Stream-stream join demo
```bash
./demo.sh join
tmux attach -t sdms-demo
```

In the CLI pane, run:
```text
load schemas/stream_join_demo.json
status
```

In the `kafka-console-producer` pane (weather_stream), type lines like:
```json
{"sensor_id":"s1","humidity":42.0,"zone":"z1"}
{"sensor_id":"s2","humidity":55.5,"zone":"z2"}
{"sensor_id":"s3","humidity":61.2,"zone":"z3"}
```

The pollution sensor is already publishing to `pollution_stream`.
Join results appear in consumer pane on topic `joined_out` when keys match and `value > 50`.

---

## 3) Stop demo

From any terminal:
```bash
tmux kill-session -t sdms-demo
```

Optional full cleanup (recommended after demo):
```bash
./demo_cleanup.sh
```

This removes:
- demo tmux session
- `data/static_tables.db`
- Kafka/Zookeeper temp state (`/tmp/kafka-logs`, `/tmp/zookeeper`)
- local Kafka distribution log files under `kafka_2.13-3.6.1/logs`

If you want to keep Kafka state/logs:
```bash
./demo_cleanup.sh --keep-kafka-state
```

---

## 4) Manual mode (without tmux)

If you prefer manual terminals, run these in separate windows:

1) Zookeeper:
```bash
cd kafka_2.13-3.6.1
bin/zookeeper-server-start.sh config/zookeeper.properties
```

2) Kafka broker:
```bash
cd kafka_2.13-3.6.1
bin/kafka-server-start.sh config/server.properties
```

3) Pollution sensor:
```bash
python -m sensors.pollution_sensor
```

4) CLI:
```bash
python -m examples.cli
```

5) Consumer:
```bash
cd kafka_2.13-3.6.1
bin/kafka-console-consumer.sh --bootstrap-server localhost:9092 --topic pollution_out
```

For stream-stream join, additionally run weather producer:
```bash
cd kafka_2.13-3.6.1
bin/kafka-console-producer.sh --bootstrap-server localhost:9092 --topic weather_stream
```

---

## 5) Troubleshooting

- If session already exists:
```bash
tmux kill-session -t sdms-demo
```

- If `tmux` missing, install it and rerun script, or use manual mode.

- If no output appears:
  - verify schema loaded in CLI (`status`)
  - verify table data exists for stream-table join demo
  - verify weather JSON is being typed for stream-stream join demo
  - check topic names shown by `status`
