Streaming Data Management System - Implementation Plan and Status

Current Snapshot
- Status: v1.1+ runtime is schema-first, with interactive deployment through CLI.
- Query model: Continuous SELECT queries only (no ad-hoc execution API).
- Join model: stream-stream INNER JOIN and stream-table INNER JOIN are both supported.
- Velocity model: count and time modes are both executed in window operator.
- Queue durability model: message queue is ephemeral-only (no persistence mode exposed).

Architecture Principles Applied
- Single-responsibility operators: filter, projection, window, aggregate, join, sink each own one stage.
- Deterministic routing: engine routes events by registered stream membership per deployed query.
- Schema-level invariants: window and velocity are global per schema for consistent query behavior.
- Explicit contracts: parser emits structured plans; engine maps plan type to concrete pipeline topology.
- Prefix-based collision handling: right-side join fields use stable prefixes (`right_` / `table_`).

Completed Work
- Execution engine updated for multi-input query routing in stream-stream joins.
- Stream-stream join operator added with processing-time bounded buffers.
- Window operator upgraded with velocity-aware triggering:
  - count mode: emit every N events
  - time mode: emit every T seconds
- CLI output schema inference improved for JOIN queries:
  - stream-stream join schemas merged with right-side collision prefixing
  - stream-table join schemas inferred from SQLite table metadata with `table_` collision prefixing
- Schema reload/replace workflow retained in registry for iterative deployment.

Testing Status
- New tests added under `tests/` for:
  - velocity semantics (`tests/test_window_velocity.py`)
  - stream-stream join operator behavior (`tests/test_stream_stream_join.py`)
  - engine routing for stream-stream join pipelines (`tests/test_engine_stream_stream_join.py`)
  - CLI JOIN schema inference (`tests/test_cli_schema_inference.py`)
- Result: `pytest -q` passes (7 tests).
